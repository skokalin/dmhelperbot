# 📅 Telegram Calendar Bot (TypeScript + MySQL + Docker)

Профессиональный Telegram-бот для планирования задач и мультирассылки напоминаний. Бот написан на **TypeScript** с использованием библиотеки **Telegraf** (v4), разделен на изолированные модули по принципам Clean Architecture и разворачивается одной командой через **Docker Compose**.

## 🚀 Ключевые фичи
*   **Модульная архитектура**: Код разбит по зонам ответственности (маршрутизаторы команд, экшенов кнопок, текстовых сценариев, конфигурация БД и сервисы). Его легко читать и масштабировать.
*   **Гибкие напоминания (как в Google Календаре)**: На шаге создания события можно выбрать сразу несколько интервалов (за 5/15/30 минут, 1/2 часа, 1/2 дня, неделю или в момент начала).
*   **Мультирассылка в чаты и каналы**: При создании напоминания бот выводит список групп и каналов, куда он был добавлен, позволяя выбрать галочками, куда именно нужно дублировать уведомление помимо личных сообщений.
*   **Синхронизация по МСК**: Время внутри Docker-контейнеров, Node.js и базы данных MySQL принудительно зафиксировано на часовой пояс `Europe/Moscow`. Крон-планировщик работает нативно по МСК.
*   **Автономный UI**: Календарь и таймпикер написаны вручную, что исключает падение контейнера из-за сломанных или заброшенных внешних npm-пакетов.

## 📁 Модульная структура проекта
```text
ts_mysql_bot/
├── src/
│   ├── config/
│   │   └── db.ts          # Пул подключений к MySQL
│   ├── keyboards/
│   │   └── inline.ts      # Сетки кнопок (календарь, часы, минуты, чаты, напоминания)
│   ├── services/
│   │   └── cron.ts        # Фоновый крон-планировщик (проверка каждую минуту по МСК)
│   ├── handlers/
│   │   ├── commands.ts    # Обработчики команд (/start, /add, /list)
│   │   ├── actions.ts     # Обработчики кликов по кнопкам (переключение шагов меню)
│   │   └── text.ts        # Обработчик текста (сохранение названия и расчет дат в БД)
│   └── bot.ts             # Главная точка входа и диспетчер
├── package.json           # Зависимости и скрипты (ts-node)
├── tsconfig.json          # Конфигурация компилятора TypeScript (папка src)
├── Dockerfile             # Инструкция сборки Node.js-окружения
├── init.sql               # Схема таблиц MySQL (events, event_reminders, chats)
└── docker-compose.yml     # Оркестратор контейнеров приложения и СУБД
```

## 💻 Локальный запуск и тестирование (WSL / Linux)

### 1. Переменные окружения
Создайте в корне проекта файл `.env` и впишите туда токен вашего бота от `@BotFather`:
```env
BOT_TOKEN=1234567890:ABCdefGhIJKlmNoPQRsTUVwXyZ
```

### 2. Запуск контейнеров
Запустите сборку проекта в интерактивном режиме, чтобы видеть логи приложения и планировщика в реальном времени:
```bash
docker compose up --build
```

### 3. Сброс базы данных
Если во время тестирования вам понадобится полностью очистить базу данных и пересоздать таблицы с нуля, выполните жесткий сброс томов Docker:
```bash
docker compose down -v
```

## 🌐 Деплой на VPS (Ubuntu 22.04 / 24.04 LTS)

### Шаг 1: Автоматическая установка Docker на чистый сервер
Подключитесь к вашему VPS по SSH и выполните команды для установки актуальной официальной версии Docker:

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y ca-certificates curl gnupg lsb-release

sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://docker.com | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

echo \
  "deb [arch=\$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://docker.com \
  \$(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Разрешаем запуск Docker без sudo (требуется перезапуск SSH-сессии)
sudo usermod -aG docker \$USER
```

### Шаг 2: Настройка BotFather (Важно!)
Для того чтобы бот корректно работал в группах и видел обычный текст (название события):
1. Напишите `@BotFather` команду `/mybots`.
2. Выберите своего бота ➡️ **Bot Settings** ➡️ **Group Privacy**.
3. Нажмите **Turn off** (должно стать `Privacy mode is disabled`).
4. Если бот уже был добавлен в чаты, удалите его оттуда и добавьте заново.

### Шаг 3: Развертывание
1. Перенесите файлы проекта на VPS (через Git или SFTP).
2. Создайте файл `.env` с вашим `BOT_TOKEN` в корневой папке на сервере.
3. Запустите проект в фоновом режиме (как демон):
```bash
docker compose up -d --build
```
*Контейнеры настроены в режим `--restart unless-stopped`, поэтому бот автоматически поднимется даже после внезапной перезагрузки самого VPS.*
